Create a DataFrame from scratch

like INSERT + SELECT * in Teradata
cell_01 · create dataframe Action 
# Raw data as a Python list — like VALUES in Teradata INSERT
data = [("Alice", "UAE", 5000),
        ("Bob",   "KSA", 7000),
        ("Carol", "UAE", 4500)]

# Column names — like column definitions in CREATE TABLE
columns = ["name", "country", "salary"]

# Create DataFrame — like a table in memory
df = spark.createDataFrame(data, columns)

# Show results — like SELECT * FROM table
df.show()
spark.createDataFrame(data, cols)Creates in-memory table from a Python list. spark is always available in Databricks — no setup needed.
df.show()ACTION — triggers actual computation and prints results. Nothing runs until an action is called.

like WHERE in Teradata

# Filter — like WHERE country = 'UAE' in Teradata
uae_df = df.filter(df.country == "UAE")
uae_df.show()
.filter(condition)Keeps only rows matching the condition. Lazy — nothing runs until .show() or another action is called.
df.country == "UAE"Column condition. Can also write col("country") == "UAE" — both work the same way.

Add a calculated column

like CASE WHEN in Teradata
cell_04 · withColumn + when Transformation 
from pyspark.sql.functions import when, col

# Add grade column — like CASE WHEN salary >= 6000 THEN 'Senior' ELSE 'Junior'
df_with_grade = df.withColumn("grade",
    when(col("salary") >= 6000, "Senior")
    .otherwise("Junior"))

df_with_grade.show()
.withColumn("name", expr)Adds a new column without touching existing ones. Non-destructive — original df unchanged.
when(condition, value)Equivalent of CASE WHEN in Teradata. Chain multiple .when() calls for multiple conditions.
.otherwise(value)The ELSE clause. What to put if no when() condition matched.
col("salary")References a DataFrame column explicitly. Needed so Python doesn't confuse it with a variable.

8

Append new rows

like INSERT INTO in Teradata
cell_08 · append rows Save 
from pyspark.sql import Row

# Create one new row — like a single INSERT in Teradata
new_employee = spark.createDataFrame([
    Row(name="David", country="UAE", salary=9000, grade="Senior")
])

# Append — adds rows without deleting existing data
new_employee.write.format("delta").mode("append").saveAsTable("employee_grades")

spark.read.format("delta").table("employee_grades").show()
.mode("append")Adds new rows — keeps all existing rows. Like INSERT INTO in Teradata.
.mode("overwrite")Replaces everything. Like TRUNCATE TABLE then INSERT. Use carefully!
Row(col=value)Creates a single record with named columns — like one row of VALUES in Teradata INSERT.
9

Delta Time Travel

not possible in Teradata — Delta superpower
cell_09 · time travel Delta 
# Travel back to version 0 — before David was added
df_v0 = spark.read.format("delta") \
    .option("versionAsOf", 0) \
    .table("employee_grades")
df_v0.show()

# Travel to version 1
df_v1 = spark.read.format("delta") \
    .option("versionAsOf", 1) \
    .table("employee_grades")
df_v1.show()
.option("versionAsOf", N)Returns the table exactly as it was at version N. Every write creates a new version automatically.
.option("timestampAsOf", "date")Alternative — travel by timestamp instead of version number.

View Delta history

automatic audit log — not available in Teradata
cell_10 · transaction history Delta 
from delta.tables import DeltaTable

# Get Delta management object for the table
delta_table = DeltaTable.forName(spark, "employee_grades")

# Show clean readable history — select only useful columns
delta_table.history().select(
    "version", "timestamp", "operation", "operationParameters"
).show(truncate=False)
DeltaTable.forName(spark, "name")Gets a management handle for the table — lets you view history, run OPTIMIZE, VACUUM etc.
.history()Returns full audit log — version, timestamp, who made the change, what operation.
.select("col1","col2")Like SELECT specific columns in Teradata — picks only the columns you want to see.
show(truncate=False)Prevents Databricks from cutting off long text in columns. Like widening columns in DBeaver.

Remove duplicates

like SELECT DISTINCT in Teradata
cell_11 · deduplicate Transformation 
# Remove exact duplicates — like SELECT DISTINCT * in Teradata
df_clean = spark.read.format("delta").table("employee_grades").dropDuplicates()

# Save clean version back — overwrite the messy table
df_clean.write.format("delta").mode("overwrite").saveAsTable("employee_grades")

spark.read.format("delta").table("employee_grades").show()
.dropDuplicates()Removes rows where ALL columns match — like SELECT DISTINCT * in Teradata.
.dropDuplicates(["id"])Removes rows sharing the same id value even if other columns differ. More targeted.

Create raw data and save Bronze layer

staging table — everything as received
Bronze — raw layer
cell_01 · bronze layer Save 
# Raw data — intentionally messy like real production data
# Bob is duplicated, Eve has null country, Frank has null salary
raw_data = [
    (1, "Alice", "UAE", 5000, "2024-01-15"),
    (2, "Bob",   "KSA", 7000, "2024-01-15"),
    (3, "Carol", "UAE", 4500, "2024-01-16"),
    (4, "David", "UAE", 9000, "2024-01-16"),
    (2, "Bob",   "KSA", 7000, "2024-01-15"),   # duplicate
    (5, "Eve",   None,  6000, "2024-01-17"),   # null country
    (6, "Frank", "UAE", None, "2024-01-17")    # null salary
]

columns = ["id", "name", "country", "salary", "hire_date"]
df_raw = spark.createDataFrame(raw_data, columns)
df_raw.show()
print(f"Total raw rows received: {df_raw.count()}")

# Save as Bronze — no changes, everything as received
df_raw.write.format("delta").mode("overwrite").saveAsTable("bronze_employees")
print(f"Bronze saved with {df_raw.count()} rows")
None in PythonBecomes NULL in Spark DataFrame — same concept as NULL in Teradata.
f"text {variable}"f-string — Python way to embed variables inside text. Like PRINT 'Rows: ' || COUNT(*) in Teradata stored procedures.
Bronze — no cleaningNever clean Bronze. It is the source of truth. Silver and Gold can always be rebuilt from it.
2

Build and save Silver layer

clean, deduplicate, fix data types
Silver — clean layer
cell_02 · silver layer Transformation 
from pyspark.sql.functions import col, to_date

# Always read Silver FROM Bronze — never from raw list
df_silver = spark.read.format("delta").table("bronze_employees")

# Step 1 — Remove duplicates by id — like SELECT DISTINCT id in Teradata
df_silver = df_silver.dropDuplicates(["id"])

# Step 2 — Remove rows with null country or salary
# like WHERE country IS NOT NULL AND salary IS NOT NULL in Teradata
df_silver = df_silver.filter(
    col("country").isNotNull() &
    col("salary").isNotNull()
)

# Step 3 — Convert hire_date string to proper date type
# like CAST(hire_date AS DATE FORMAT 'YYYY-MM-DD') in Teradata
df_silver = df_silver.withColumn(
    "hire_date",
    to_date(col("hire_date"), "yyyy-MM-dd")
)

df_silver.show()
print(f"Silver has {df_silver.count()} clean rows")

# Save Silver — separate step from transformation intentionally
df_silver.write.format("delta").mode("overwrite").saveAsTable("silver_employees")
col("x").isNotNull()Checks column is not null. Use in .filter() to remove null rows. Like IS NOT NULL in Teradata.
& between conditionsAND operator for column conditions in PySpark. Use | for OR.
to_date(col("x"), "format")Converts string to proper date type. Format must match the string — "yyyy-MM-dd" for "2024-01-15".
Save separately from transformIf save fails you know transform worked. If transform fails you never save bad data. Always separate these steps.
3

Build and save Gold layer

business aggregations for dashboards
Gold — business layer
cell_03 · gold layer Transformation 
from pyspark.sql.functions import count, avg, max, round

# Always read Gold FROM Silver — never from Bronze directly
df_gold = spark.read.format("delta").table("silver_employees")

# Aggregate by country — like GROUP BY with multiple aggregates in Teradata
df_gold_summary = df_gold.groupBy("country").agg(
    count("id").alias("total_employees"),          # COUNT(*)
    round(avg("salary"), 2).alias("avg_salary"),   # ROUND(AVG(salary),2)
    max("salary").alias("max_salary")              # MAX(salary)
)

# Sort by most employees first — like ORDER BY total_employees DESC
df_gold_summary.orderBy("total_employees", ascending=False).show()

# Save Gold
df_gold_summary.write.format("delta").mode("overwrite").saveAsTable("gold_employee_summary")
.agg(func1, func2, ...)Multiple aggregations in one go. Like SELECT COUNT(*), AVG(), MAX() GROUP BY in Teradata.
count("id").alias("name")Count rows and rename result column. .alias() = AS in Teradata SQL.
round(avg("salary"), 2)Average salary rounded to 2 decimal places. Like ROUND(AVG(salary),2) in Teradata.
orderBy("col", ascending=False)Sort descending. ascending=True (or omit) for ascending order.
