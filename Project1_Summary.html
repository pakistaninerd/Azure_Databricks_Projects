Medallion Architecture
Bronze — Raw Layer

Stores everything exactly as received — duplicates, nulls, wrong formats included. Never modify this layer. It is your safety net. If Silver or Gold break, you always rebuild from Bronze.

df_raw.write.format("delta").mode("overwrite").saveAsTable("bronze_employees")
↓
Silver — Clean Layer

Remove duplicates, handle nulls, convert data types. This is where raw data becomes trustworthy. Data engineers own this layer.

df_silver = df_silver.dropDuplicates(["id"])
df_silver = df_silver.filter(col("country").isNotNull() & col("salary").isNotNull())
df_silver = df_silver.withColumn("hire_date", to_date(col("hire_date"), "yyyy-MM-dd"))
↓
Gold — Business Layer

Pre-aggregated summaries for dashboards, reports and executives. Business users only ever touch Gold. Built from Silver — never from Bronze directly.

df_gold = df_silver.groupBy("country").agg(
  count("id").alias("total_employees"),
  round(avg("salary"), 2).alias("avg_salary"),
  max("salary").alias("max_salary")
)

Delta Lake — Key Concepts
Time Travel

Query any historical version of a table. If data gets corrupted, recover instantly without backups.

spark.read.format("delta")
 .option("versionAsOf", 0)
 .table("bronze_employees")
Transaction History

Every change is automatically logged — who changed what, when, which version. Zero configuration needed.

delta_table.history()
 .select("version", "timestamp", "operation")
 .show()
ACID Transactions

Atomicity — all or nothing. Consistency — always valid. Isolation — no conflicts. Durability — survives failures.

Write Modes

append — adds rows, keeps existing. Like INSERT.
overwrite — replaces everything. Like TRUNCATE + INSERT.

Tips to Remember
★ Always terminate your cluster after every session. Auto-termination is set to 30 min but manually terminate too. A 2hr session costs roughly $0.28 — leaving it running overnight costs much more.
★ Run All cells after cluster restart. Variables like df_silver live in cluster memory — they disappear when cluster stops. Always Run All from top before adding new cells in a new session.
★ Transformations vs Actions. filter(), groupBy(), withColumn() are lazy — nothing runs. Only show(), count(), save() trigger actual execution on your cluster.
★ Bronze → Silver → Gold always. Gold reads from Silver. Silver reads from Bronze. Never skip layers. Never read Gold from Bronze directly.
★ dropDuplicates(["id"]) vs dropDuplicates(). Without column list — removes only 100% identical rows. With column list — removes rows sharing that column value even if other columns differ.
★ Nobody memorizes PySpark syntax. Senior engineers Google and use AI daily. What you memorize is the concept — filter, group, aggregate, join. Syntax comes naturally with practice after week 3-4.
